# SkillForge — Audit Report (Round 16)

*Audited: 2026-06-30 · Findings verified against the codebase before recording.*

> Follow-up to rounds 1–15. This is the **nonce-based CSP** round — the last open *pure-security*
> item, carried since round 4 and deferred through the static-CSP rounds (R12/R13). It removes
> `'unsafe-inline'` (and `'unsafe-eval'`) from `script-src`, the directive that otherwise lets an
> injected inline `<script>` (reflected/stored XSS) execute despite a CSP being present.
>
> Scope: `next.config.js` + new `middleware.ts` + new `app/lib/csp.ts` + new test + a render-mode
> change on the root layout and play route. **No Firestore rules change, no game-artifact change.**
> Deploy = rebuild the frontend image on the Pi (the prod compose runs Next directly, so the
> middleware executes with no nginx dependency).

---

## HIGH — Fixed this round (XSS surface)

### R16-1. `script-src 'unsafe-inline'` → per-request nonce — ✅ FIXED
**Files:** `app/lib/csp.ts` (new), `middleware.ts` (new), `next.config.js`

**Verified:** the static CSP (R12/R13) shipped `script-src 'self' 'unsafe-inline' 'unsafe-eval'`
on every app route (`next.config.js:44`). `'unsafe-inline'` defeats the core purpose of a CSP —
any injected `<script>…</script>` runs. It was present only because a `next.config.js` header
cannot mint a per-request nonce and Next.js App Router emits ~258 inline RSC streaming scripts
that need to be allowed.

**Fixed:** a new `middleware.ts` mints a per-request nonce (`generateNonce()`, Web Crypto so it
runs in the **Edge runtime**) and sets `Content-Security-Policy` on both the request and the
response. Next.js reads the nonce from the request CSP header and **automatically stamps it onto
every framework `<script>`** it emits during SSR — no manual threading of the 258 tags. The CSP
now ships `script-src 'self' 'nonce-<value>'`: **`'unsafe-inline'` and `'unsafe-eval'` are gone**
from `script-src`. The policy string is built by the pure, unit-tested `buildCsp(nonce)`
(`app/lib/csp.ts`), the single source of truth; the old Rule B was removed from `next.config.js`.

### R16-2. Game-iframe exclusion preserved via the middleware matcher — ✅ FIXED
**File:** `middleware.ts`

The old CSP was scoped with `source: '/:path((?!games/).*)'` so the pre-built game iframes under
`public/games/*` (e.g. `geoguessr-clone`'s sprawling Google Maps domains) stayed unconstrained.
The middleware `matcher` reproduces this exactly — `'/((?!api|_next/static|_next/image|games|favicon.ico).*)'`
plus a `missing` prefetch guard so a cached prefetch can't pin a stale nonce. Verified live:
`/games/2048/` carries **no** app CSP header.

### R16-3. App routes forced dynamic so the runtime nonce reaches every script — ✅ FIXED
**Files:** `app/layout.tsx`, `app/play/[gameId]/page.tsx`

A per-request nonce cannot be baked into statically prerendered HTML — those scripts would lack
the runtime nonce and be CSP-blocked. `app/play/[gameId]/page.tsx` previously prerendered all game
host pages via `generateStaticParams` (now removed); both the root layout and the play route now
export `dynamic = 'force-dynamic'`. This is the accepted, inherent tradeoff of nonce-based CSP
(no static prerender) and as a side effect removes the long-standing build-time Firebase-key
prerender warning on `/auth/action` & `/_not-found` (those routes no longer prerender). Build
output confirms every app route is now `ƒ (Dynamic)`.

## Fresh sweep — checked, no new finding
- **`style-src 'unsafe-inline'` kept** — nonces do not cover `style=""` attributes, and React Hot
  Toast + ~9 components use `style={{}}` props. Removing it is a separate refactor (CSS modules /
  Tailwind), explicitly out of scope, same stance as R13. Inline-style XSS is a far weaker vector
  than inline-script.
- **`'unsafe-eval'` removed too** — verified no app-route runtime needs eval in production (Firebase
  JS SDK is fine; the games that use eval are iframed under `/games/*`, excluded from the matcher).
  If a future dependency needs it, add `'unsafe-eval'` back to `script-src` in `buildCsp` only —
  the headline win (`'unsafe-inline'` gone) is independent.
- **Baseline headers unchanged** — `X-Frame-Options: SAMEORIGIN`, `X-Content-Type-Options`,
  `Referrer-Policy`, HSTS stay in `next.config.js` Rule A (`/:path*`), still covering `/games/*`.
- **Firebase/Google allowlists preserved** — `frame-src *.firebaseapp.com` (signInWithPopup helper
  iframe), `connect-src` Firestore/Auth/Storage hosts, and the `img-src` avatar allowlist (R12) are
  copied verbatim. Verified live in the served header.

## Verification
- `npm run build` — green; build output lists every app route as `ƒ (Dynamic)` and registers
  `ƒ Proxy (Middleware)`; the prior `/auth/action` / `/_not-found` prerender warning no longer fires.
- `npm test` — **58/58** green (53 prior + 5 new `tests/csp.test.ts`): `buildCsp` keys `script-src`
  to the nonce and asserts `'unsafe-inline'`/`'unsafe-eval'` absent from it; required directives
  (`default-src`, `object-src 'none'`, `base-uri`, `frame-ancestors`) and the Firebase/Google
  connect+frame and avatar img-src allowlists are preserved; `style-src` retains `'unsafe-inline'`;
  `generateNonce()` returns distinct base64 values.
- **Live `next start`** (verified this round): `/login/` → 200 with
  `script-src 'self' 'nonce-…'` and **no** `'unsafe-inline'`/`'unsafe-eval'`; served `<script>`
  tags carry matching `nonce="…"`; each request gets a **distinct** nonce; `/games/2048/` carries
  **no** CSP header. (Edge-runtime regression caught + fixed during impl: `node:crypto`/`Buffer`
  aren't available in middleware — switched `generateNonce` to Web Crypto + `btoa`.)
- **Manual E2E to run post-deploy:** DevTools on an app route shows the nonce'd CSP and **zero**
  CSP violations through load + hydration + navigation; Google sign-in popup completes; geoguessr
  Street View renders (iframe excluded).
