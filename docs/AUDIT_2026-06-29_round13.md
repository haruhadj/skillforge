# SkillForge — Audit Report (Round 13)

*Audited: 2026-06-29 · Findings verified against the codebase and a live `next start` before recording.*

> Follow-up to rounds 1–12. Round 12 shipped a partial CSP (`default-src`, `object-src`,

## MEDIUM — Fixed this round

### R13-1. CSP completed: `script-src` / `style-src` / `font-src` / `connect-src` added — ✅ FIXED
**File:** `next.config.js`

**Verified (empirically, against a `next start` build):**
- The prerendered HTML contains **258 inline `<script>` tags** — Next.js App Router emits
  inline RSC streaming scripts. A `script-src` without `'unsafe-inline'` (and no nonce)
  renders the entire app blank. The plan's `'self' 'unsafe-eval'` would have broken it.
- Firebase SDK JS is npm-bundled (no external `<script>` tags); `next/font/google` self-hosts
  fonts at build time (no runtime `fonts.googleapis.com` fetch).
- App backend (`/api/*`) is rewritten **server-side**; the browser only ever connects
  same-origin. Socket.IO paths (`/chess-ws/`, `/tictactoe-ws/`, `/chroma-memory-ws/`) are
  same-origin in prod (nginx-proxied). Client→cloud connections are Firestore/Auth/Storage
  over `*.googleapis.com` (REST + gRPC/WebChannel, incl. `wss://`).
- Inline `style={{}}` props are used throughout (animation delays, gradients, dynamic
  widths) plus React Hot Toast — `style-src` needs `'unsafe-inline'` short of a nonce round.

**Fixed:** added four directives to the app-route CSP:
```
script-src 'self' 'unsafe-inline' 'unsafe-eval'
style-src  'self' 'unsafe-inline'
font-src   'self'
connect-src 'self' https://*.googleapis.com wss://*.googleapis.com
            https://*.firebaseio.com wss://*.firebaseio.com https://accounts.google.com
```
`'unsafe-inline'`/`'unsafe-eval'` are documented trade-offs: required by Next.js App Router
(inline streaming scripts) and the pre-built game bundles (eval). `'self'` still blocks
**external-origin** script/style injection — the most common XSS exfiltration vector — and
establishes the directives so the deferred nonce round only has to tighten them.

### R13-2. `frame-src 'self'` (shipped in R12) would block Firebase Google sign-in — widened — ✅ FIXED
**File:** `next.config.js`

**Verified:** `AuthContext.tsx:36` uses Firebase's built-in `signInWithPopup(auth, new
GoogleAuthProvider())`. The Firebase JS SDK loads a hidden helper **iframe** from the
authDomain (`skillforge-7a058.firebaseapp.com/__/auth/iframe`) to manage auth state. R12's
`frame-src 'self'` would have blocked that iframe — silently breaking Google popup sign-in
once R12 deployed. (The custom OAuth broker via `signInWithCustomToken` is unaffected.)

**Fixed:** `frame-src 'self' https://*.firebaseapp.com https://accounts.google.com` — allows
the Firebase auth iframe and the Google OAuth surface, while still blocking arbitrary
third-party frames. This retroactively repairs the latent R12 regression.

### R13-3. Strict CSP scoped to app routes; game iframes excluded — ✅ FIXED
**File:** `next.config.js`

**Verified:** Next.js `headers()` with `source: '/:path*'` applies to the static game HTML
under `public/games/*` too. The `geoguessr-clone` game iframe loads the Google Maps JS API,
which pulls in a sprawling, hard-to-enumerate set of Google domains (Street View tiles,
`khms*`, `streetviewpixels-pa`, `maps.gstatic.com`, …). Other games are self-contained Vite
bundles (relative paths only) but use `eval`.

**Fixed:** the strict CSP rule now uses `source: '/:path((?!games/).*)'` (path-to-regexp
negative lookahead), so it applies to **app routes only**. Game iframes — trusted, pre-built
artifacts we shipped, sandboxed in iframes — are left unconstrained rather than risk breaking
gameplay for little gain. They retain clickjacking protection via the baseline
`X-Frame-Options: SAMEORIGIN`, which stays on `/:path*` (all paths). The four R4 baseline
headers were split into their own `/:path*` rule so they still cover `/games/*`.

**Live verification (`next start`, port 3300):**
- `GET /login/` → full CSP present (all 9 directives) **and** `X-Frame-Options`.
- `GET /games/2048/` → **CSP header count 0**, `X-Frame-Options` still present. Exclusion works.

## Fresh sweep — checked, no new finding
- `connect-src 'self'` covers dev HMR websocket (same-origin) and the `/api/*` rewrites;
  `'unsafe-eval'` covers React Refresh in dev. Dev mode unaffected. Clean.
- `*.googleapis.com` (+ `wss://`) covers Firestore streaming, Auth (identitytoolkit,
  securetoken), and Storage uploads. `*.firebaseio.com` reserved (RTDB unused). Clean.
- Game bundles other than geoguessr load scripts only from relative/local paths — would be
  fine under the strict CSP, but are excluded uniformly with geoguessr for simplicity. Clean.
- `img-src` allowlist (R12) unchanged and still enforced on app routes. Clean.
